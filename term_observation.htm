Observation

    Climate/meteorological data for the watershed being modeled. This input drives the model.  The data is read from a data file with the file extension obs, (*.obs).

    The usual modeling time step is a half hour or one hour. Daily data starts at the first interval of the day e.g. 00:30 or 01:00.  The end of a day is either 24:00 or 00:00 of the next day. Gaps in the data must be filled with synthesized data to allow the model to run.

    Module driving inputs are Observations or Input Variables.

Function Observations

    The root observation is indicated by the observation name, e.g. 't'.   Function observations are denoted by an extension beginning with an underscore to the observation name, e.g. 't_Avg'.  Available observation functions are:

Average _Avg daily average value.
Minimum _Min daily minimum value.
Maximum _Max daily maximum value.
Daily Sum _Sum sum of daily interval values.
Positive _Pos sum of daily interval values that are greater than zero.
Total _Tot total of interval values over entire run period
First _First displays data once at the beginning of the day.
Last _Last displays data once at the end of the day.
W_to_MJ _WtoMJ the data is converted from Watts to Mega Joules per time interval.
MJ_W _MJtoW the data is converted from mega Joules per time interval to Watts.
